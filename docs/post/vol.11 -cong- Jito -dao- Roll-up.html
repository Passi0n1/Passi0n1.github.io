vol.11 从 Jito 到 Roll-up

从 Jito 到 Roll-up

引言

在区块链技术的持续发展过程中，不同的协议和机制不断涌现，以应对网络扩展性、性能、成本等方面的挑战。本文通过对 Jito 的了解学习，延伸至 空间竞价、zk-Rollup 和 Optimistic Rollup（简称 OP Rollup）等技术，全面探讨这些技术如何协同作用，提升区块链的吞吐量、效率与安全性。

一、Jito 协议与 MEV 奖励机制

Jito 是一个基于 Solana 区块链的质押协议，旨在通过 MEV（最大化可提取价值）奖励机制为质押者带来额外收益。该协议的运作流程如下：

运行机制

  • Jito 质押池： 用户将 SOL 代币 存入 Jito 质押池，并获得代表存入 SOL 的代币——JitoSOL。
  • 委托验证者： 存入的 SOL 代币会被委托给具备 MEV 能力的验证者。这些验证者具备拍卖区块空间以赚取 MEV 奖励的能力。
  • MEV 奖励发放： 当验证者成功提取 MEV 奖励时，一部分奖励会流回 Jito 质押池，提升参与者的年化收益率（APY）。

MEV 奖励

MEV 的产生

  • 验证者通过观察交易池（Mempool）中的交易，找到潜在的套利机会或高价值交易。
  • 验证者将这些交易插入到区块中合适的位置，以获得额外收益。

MEV 的拍卖

  • Jito 使用一种拍卖机制，允许交易发送方（比如机器人）为区块空间竞价。
  • 验证者从最高出价者那里选择交易并收取费用。

MEV 奖励指的是通过优化区块打包顺序或交易执行方式，从而为验证者带来额外的利润。这种机制的核心在于 抢跑等策略找到可盈利的交易进行三明治的攻击操作、或者优先打包，或者拍卖区块空间、来通过控制区块空间的排序来获得比常规交易更多的收益。

二、空间竞价：区块空间与交易打包顺序

在 Jito 和其他类似协议中，区块空间被视为一个稀缺资源，参与者通过竞价来争夺区块内的位置。区块空间，在此语境下，指的是交易在区块链中被打包的顺序，这直接影响交易的执行成本与执行顺序。

为何称之为区块空间竞价？

在考虑其实际作用场景，发现更多的是被打包的优先顺序，那么为什么不称之为交易优先顺序竞价？ 有这样几个原因

  • 区块空间更符合gas、存储等区块链空间化的概念
  • 交易优先顺序竞价潜意识透漏出的以gas为单一维度去衡量的，而忽略合约调用、以及在rollup场景下的被压缩的数据的存储等概念

三、zk-Rollup 与 Optimistic Rollup

在区块空间中了解到了一个rollup，之前也有接触，但是总是有一些疑问，这里更加明确一下：

zk-Rollup：基于零知识证明的扩展性

zk-Rollup 是一种 Layer 2 扩展解决方案，通过批量处理交易并使用零知识证明（ZK-SNARKs）来验证交易的合法性。它通过将大量交易压缩成一个较小的数据包，显著提高区块链的吞吐量。zk-Rollup 的优势包括：

  • 低延迟： zk-Rollup 能够快速处理和确认交易，并且具备极高的交易效率。
  • 安全性： 由于采用零知识证明，zk-Rollup 可以确保交易的真实性而无需逐个验证。
  • 数据压缩： 交易数据通过压缩后存储在 Layer 1 上，减轻了主链的负担。

Optimistic Rollup：假设交易合法的扩展机制

与 zk-Rollup 不同，Optimistic Rollup 假设大多数交易是合法的，因此允许交易在未经验证的情况下先行处理。交易的合法性在 挑战期 内接受审查，任何用户都可以提出挑战并提交证据。若挑战成功，相关交易将被回滚。其特点包括：

  • 延迟确认： Optimistic Rollup 的交易在初步处理后进入挑战期，可能需要等待一段时间才能最终确认。
  • 灵活性： 由于交易默认合法，系统能够更快地处理大规模交易，但也依赖于挑战期的机制来保证安全性。

对比

  • zk-Rollup 和 Optimistic Rollup 都旨在解决 Layer 1 的扩展性问题，但它们的核心差异在于验证方式。zk-Rollup 采用零知识证明，在交易确认时即确保合法性；而 Optimistic Rollup 则假设交易合法，并通过挑战机制验证。
  • ETH主链通过逐笔交易验证和全球节点共同参与保障安全性。但是代价就是吞吐量太小。 layer2为了解决这一问题设计出了zk 和 OP 来批量处理，大幅提高吞吐量。
  • 无论是zk还是OP的roll up 都是在本链存储一份数据，而将压缩或者证明部分打包在主链，从而提高吞吐量。

四、Layer 1 与 Layer 2：为什么 Layer 1 不使用 zk 或 OP？

在传统的 Layer 1 区块链中，如以太坊，所有交易都必须逐笔验证，这导致其扩展性受到限制。虽然 zk-Rollup 和 Optimistic Rollup 能够有效提高 Layer 2 的吞吐量，但 Layer 1 却不能简单地直接采用这两种技术。原因主要包括：

  • Layer 1 安全性： Layer 1 需要保证网络的安全性和去中心化，这要求每笔交易都经过全球节点的逐笔验证。相比之下，Layer 2 可以在不影响安全性的前提下通过聚合和压缩交易数据来提高效率。
  • 数据存储限制： Layer 1 区块链上的存储空间有限，而 Rollup 技术通过将交易数据压缩并仅将验证信息上传到 Layer 1，有效减少了存储需求。
